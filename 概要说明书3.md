## 基于地图可视化的 **AI** 个性化旅游路线生成系统 概要说明书 

## 1. 引言 

## 1.1 编写目的 

本文档明确基于地图可视化的 AI 个性化旅游路线生成系统的功能需求与非功能 需求，作为后续系统设计、开发、测试以及项目验收的依据，统一开发团队对 需求的理解，减少沟通偏差。 

## 1.2 项目背景 

随着自助游、个性化出行的需求持续增长，传统固定旅游路线无法满足不同用 户的偏好差异（如预算、出行时长、兴趣类型、体力情况等）。本项目结合 AI 大语言模型对用户需求的理解能力，搭配地图可视化的直观展示，为用户快速 生成符合个人需求的专属旅游路线，解决游客规划行程耗时费力的痛点。 

## 1.3 定义、缩写词与缩略语 

AI 大语言模型：具备自然语言理解与文本生成能力的人工智能模型，本项目用 

## 于解析用户出行需求、生成个性化路线排序。 

地图可视化：将地理空间信息（景点位置、路线）以交互式地图的形式直观展 示给用户。 

B/S 架构：浏览器/服务器架构，用户无需下载安装客户端，通过浏览器即可访 问使用。 

2. 总体需求描述 

- 2.1 目标用户 

本系统目标用户为： 

计划自助出游的个人游客与家庭出游群体 

想要探索小众目的地的旅行爱好者 没有提前规划时间、需要快速生成行程方案的短途出行用户 

## 2.2 运行环境 

用户端：运行于主流 PC 端浏览器（ Chrome 、Edge 、 Firefox 等），支持 

## Windows/macOS 系统 

后端服务：部署于云服务器，依赖第三方开放 API 提供地图与 AI 能力 

数据存储：使用关系型数据库存储结构化景点与用户数据 

2.3 约束与限制 

本项目为学生课程项目，开发周期有限，仅覆盖国内景点的路线生成，暂不支 持出境游需求 

地图与 AI 能力依赖第三方开放 API，功能可用性受第三方接口服务状态影响 

初始版本仅支持 PC 端访问，暂不做移动端深度适配

3.3 非功能性需求 

性能需求：从用户提交需求到生成路线，整体响应时间不超过 20 秒；单张地图 加载完成时间不超过 3 秒 

易用性需求：核心生成流程步骤不超过 3 步，界面布局清晰，新手用户可在 1 分钟内理解操作逻辑 

兼容性需求：兼容 Chrome 、 Edge 、 Firefox 浏览器最新 3 个版本，适配 

1920*1080 及以上分辨率 

数据安全性需求：用户手机号信息加密存储，管理端接口做权限验证，防止未 授权访问 

4. 需求优先级划分 

优先级说明 

- 高必须实现，为核心功能，不实现则项目不完整 

- 中计划实现，在核心功能完成后开发，若开发时间不足可适当裁剪 

- 低优先度最低，项目开发周期充足时实现，否则项目验收时可缺失 

5. 验收标准 

所有优先级为高的需求必须 100%实现，满足需求描述中的功能要求，核心流 

程无严重 bug 

非功能性需求中的性能与易用性要求必须满足，响应时间与操作步骤符合标准 地图展示、路线生成、数据管理核心流程运行稳定，无崩溃、数据丢失问题。
