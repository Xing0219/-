## 基于地图可视化的 **AI** 个性化旅游路线生成系统 详细设计说明书 

## 1. 引言 

## 1.1 编写目的 

本文档基于前期需求分析说明书，对系统进行整体架构设计与模块划分，明确 各模块功能、数据结构与交互逻辑，作为后续详细设计与代码实现的直接依据， 供开发团队成员参考使用。 

## 1.2 项目背景 

项目核心目标是解决自助游客规划行程效率低、个性化适配不足的痛点，采用 B/S 架构开发，结合第三方地图开放 API 与 AI 大模型 API，实现用户需求输入 

→AI 路线生成→地图可视化展示的完整流程。 

## 1.3 定义、缩写词与缩略语 

同需求分析说明书，核心术语如下： 

术语/缩写 定义说明 

B/S 浏览器/服务器架构，前端运行于浏览器，后端提供接口服务 

MVC 模型-视图-控制器架构模式，用于后端业务分层解耦

## 2.3 核心处理流程 

## 2.3.1 路线生成核心流程 

用户在前端填写出行需求（可视化筛选+自然语言补充），点击「生成路线」按 钮提交给后端 

后端接收请求后，根据用户填写的目的地城市、预算、兴趣标签，从数据库筛 选符合条件的候选景点列表 

后端将用户原始需求+候选景点信息整理为标准化提示词，调用 AI 大模型 API 

请求生成结构化路线 

AI 返回结构化的按天行程数据后，后端根据每个景点的坐标，调用地图 API 计 算景点间交通路线与时间，补充完善行程信息 

后端将完整的行程数据+景点坐标+路线信息返回给前端 

前端接收数据后，渲染文字行程列表，并在地图上标注景点、绘制每日行进路 线，完成展示 

## 2.3.2 景点数据管理核心流程 

管理员登录管理端后，可通过两种方式新增景点：手动单个录入，或批量导入 

Excel 

单个录入/批量导入时，系统自动校验必填字段（名称、城市、坐标、价格）是 

否完整，不完整则提示错误 

校验通过后，数据保存到数据库的景点表中，后续路线生成时即可被筛选调用 用户提交景点信息错误反馈后，管理员可在管理端查看反馈，修改对应景点数 据，修改完成后标记反馈为已处理，数据实时生效

4. 模块详细设计说明 

## 4.1 前端各子模块职责 

用户身份认证子模块：负责手机号验证码登录、登录状态保存、过期自动重登， 未登录用户限制保存路线等功能 

出行需求输入子模块：渲染可视化筛选控件（天数选择、预算选择、标签多 选），同时提供自由文本输入框，收集完整需求后格式化提交给后端 

AI 路线生成调度子模块：处理生成请求的提交与等待逻辑，提交后显示加载动 

画，接收返回结果后通知地图与行程模块渲染 

交互式地图可视化子模块：初始化高德地图实例，负责景点标注添加、弹窗信 息渲染、不同颜色路线绘制、地图视角适配、平移缩放交互 

行程信息展示与导出子模块：按天渲染结构化文字行程，提供复制纯文本功能， 展示行程总览信息 

我的路线管理子模块：展示用户已保存的路线列表，提供重命名、删除功能， 点击列表项加载对应路线的地图与文字信息 

4.2 后端各子模块职责 

基础 API 接口子模块：统一处理跨域、请求日志、权限验证、异常捕获，返回

景点数据筛选子模块：根据用户需求（目的地、预算、标签）构造数据库查询 条件，筛选出符合要求的候选景点列表 

AI 提示词构造与结果解析子模块：将用户需求与候选景点拼接为符合 AI 大模型 

要求的提示词，要求 AI 返回固定格式的 JSON，接收 AI 返回后解析为结构化行 程数据 

路线信息持久化子模块：处理用户保存路线请求，将路线数据序列化为 JSON 

## 存储到数据库，提供查询、删除接口 

用户反馈处理子模块：处理用户反馈提交，保存反馈到数据库，供管理员后续 处理

5. 系统出错处理设计 

## 5.1 异常信息与处理策略 

异常场景 异常信息提示 处理策略 

AI 大模型 API 调用超时/失败 "当前 AI 服务暂时繁忙，请您 1 分钟后重试"记 

录错误日志，返回用户提示，不生成路线 

第三方地图 API 调用失败 "获取路线数据失败，请您刷新页面重试" 保留已 生成的文字行程，仅地图展示失败，提示用户 

用户未登录访问需要登录的接口 "请先登录后再进行此操作" 返回 401 状态 

码，前端自动跳转登录页 

管理员权限验证失败 "您没有权限访问此功能" 返回 403 状态码，跳转管 理登录页 

请求参数格式错误 "提交的信息不完整，请检查后重新提交" 拒绝请求，提 示用户检查输入 

景点数据不存在 "该景点信息已不存在，请刷新页面后重试" 返回错误提示 引导用户重新操作 

## 5.2 系统维护设计 

所有异常错误都记录后端日志，包含错误时间、错误类型、请求信息，方便开 发调试排查问题 

数据库每日自动备份，防止数据丢失 

景点数据更新后自动缓存，减少数据库查询次数，提升接口响应速度
